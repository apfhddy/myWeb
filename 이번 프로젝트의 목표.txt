이번 프로젝트의 목표

--프로젝트 방향성
java 라이브러리들을 쉽게 검색 ( 리턴값이 지정 클래스 필터 또는 상속 ) 및 데이터 테스트를 통한 이해
아이들이 봐도 이해돼야함 


--구현 포인트
리액트를 사용한 spa 싱글페이지 구현 및 스프링 부트 연동
Rest api를 사용하여 CRUD
팩키지,클래스의 관계를 추상화 시키기 위한 Oracle DB사용
Rest api를 통한 CUD는 Redis를 사용한 수정 권한 키인증 절차 ( 인메모리 데이터베이스 속도로 레디스 )
카페 24를 통한 도메인 취득후 공인 IP연결 포트포워딩을 통한 배포
서버는 GUI가 아닌 CLI환경으로 리눅스 CLI환경으로 구동 ( AWS또한 리눅스 서버 이기에 CLI 배포환경 적응하기 )
백엔드 서버는 스프링 부트 적응을 위한 부트 사용 
리액트와 스프링 부트 연동 공부
